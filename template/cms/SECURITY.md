# セキュリティガイド

## 🔒 実装済みセキュリティ機能

### 認証・認可
- **パスワードハッシュ化**: PHP `password_hash()` + `password_verify()` 使用
- **セッション管理**: セキュアなセッション設定、定期的なID再生成
- **ログイン試行制限**: ブルートフォース攻撃防止

### 攻撃対策
- **CSRF保護**: 全フォームでトークン検証実装
- **XSS対策**: `htmlspecialchars()` による出力エスケープ
- **ファイルアップロード**: 画像形式・サイズ・MIME型検証
- **セキュリティヘッダー**: X-Frame-Options, X-XSS-Protection など

### データ保護
- **入力値検証**: 全ての入力値を適切にサニタイズ
- **ディレクトリトラバーサル防止**: パス検証実装
- **直接アクセス防止**: .htaccess による保護

## ⚠️ セキュリティ上の注意点

### 本番環境導入前チェックリスト

メンテナーに直接連絡してください。

## 🛠 セキュリティ設定のカスタマイズ

### 高セキュリティ環境向け設定

#### IPアクセス制限の有効化
```php
// admin/security_check.php
// 以下の行のコメントを外す
checkIPAccess($ALLOWED_IPS);
```

#### セッションタイムアウトの短縮
```php
// security_config.php
$SESSION_TIMEOUT = 1800; // 30分に短縮
```

#### より厳格なファイル制限
```php
// security_config.php
$MAX_FILE_SIZE = 2 * 1024 * 1024; // デフォルト5MBから2MBに制限
$ALLOWED_IMAGE_TYPES = ['image/jpeg']; // デフォルト3形式からJPEGのみ許可
```

## 🚨 インシデント対応

### 不正アクセスが疑われる場合

1. **即座に実行**
   - 管理画面パスワードの変更
   - 全てのセッションを無効化（users.json の一時移動）

2. **調査・対応**
   - サーバーログの確認
   - アップロード済みファイルの検査
   - データベース（news.json）の整合性確認

3. **復旧作業**
   - 信頼できるバックアップからの復元
   - セキュリティ設定の見直し
   - 監視体制の強化
